Maestro Performance Reports: Help

Protocol Performance Reports

These reports compares the maximum recorded metrics for each protocol supported by the Software Under Test (SUT). This comparison is made regardless of the system/test configuration as it assumes that the configuration variations for the SUT and the test conditions are limited.

Contended Performance Reports

These reports compares the maximum recorded metrics for a protocol under destination contended conditions. This means that the data is sent through the SUT using a limited set of queues. For example, it can be used to compare the scalability of the product when 100, 10 and/or 1 connections are exchanging data via a single queue.

Destination Scalability Reports

These reports compares the maximum recorded metrics for a protocol under destination contended conditions. This means that the data is sent through the SUT using a limited set of queues. For example, it can be used to compare the scalability of the product when 100 connections are exchanging data via 1, 10 or 100 queues.

Performance By Configuration

This report compares the performance of the SUT with different test configurations/conditions. For example, it can be used to compare the product performance with different operating system versions.

Performance Delta

This report compares the SUT performance against multiple different configurations and parameters of the same SUT version. It is a super set of all other reports available and can be used to find, for example, what configuration is mostly affected by an specific set of test parameters or conditions (ie.: what configuration is most affected by a given patch or OS version). Because the data set can be very large, the reports are available as CSV files that can be downloaded an imported into Libre Office or any other spreadsheet sheet software.

The reports are composed of 2 data sets:
  • The Left Hand Side (LHS) fields contains the baseline configuration/parameters.
  • The Right Hand Side (RHS) fields contains the configuration/parameters you want to compare with.

    • Each data set contains the following fields:
  • test_id: test id.
  • sut_id: sut id.
  • sut_tags: sut tags are used to tag a SUT with details related to its base configuration (ie.: AIO or NIO journal).
  • test_tags: test tags are used to tag a test with test-specific details (ie.: whether spectre patches were present, RHEL version, etc etc).
  • sut_name: SUT name.
  • sut_version: SUT version.
  • test_valid: whether the test was not invalidated by errors, SUT bugs or other problems. This should almost always be true.
  • env_resource_role: whether the data is for a sender or receiver
  • connection_count: connection count on the node.
  • limit_destinations: the number of destinations used in the test.
  • message_size: the message size in bytes.
  • messaging_protocol: the messaging protocol.
  • variable_size: whether size variability was applied to the messages.
  • test_rate_geometric_mean: rate geometric mean.
  • test_date: test date.

    • The performance difference between configurations is available on the diff field which contains the performance delta between the LHS and RHS data sets.
